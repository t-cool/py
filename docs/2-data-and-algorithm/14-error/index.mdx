# コンピュータが苦手な計算（誤差：ごさ）

## コンピュータだって間違える？

コンピュータは計算が大得意ですが、実はどうしても少しだけ「ズレ」が出てしまうことがあります。このズレを **「<ruby>誤差<rt>ごさ</rt></ruby>」** とよびます。

どうしてズレちゃうのか、その理由を見てみましょう。

## 1. ぴったり表せない。「丸め誤差」

たとえば、10÷3 の答えは 3.3333... とずっと続きますね。コンピュータは<ruby>無限<rt>むげん</rt></ruby>に数字を覚えられないから、どこかで切り捨ててしまいます。

また、コンピュータは「2<ruby>進数<rt>しんすう</rt></ruby>」で計算しているから、人間にとってかんたんな「0.1」も、実はコンピュータの中ではぴったり表せないのです。

<ViewSource path="/error/rounding_error.ipynb" />

## 2. 大事な数字が消えちゃう。「桁落ち（けたおち）」

すごく近い数字どうしを引き算すると、大事なところが消えてしまうことがあります。

たとえば、`1.234567` から `1.234566` を引くと、答えは `0.000001` ですね。
もともとはたくさん数字があったのに、残ったのは `1` だけになってしまいました。これが「桁落ち」です。

<ViewSource path="/error/cancellation.ipynb" />

## 3. 小さな数字が無視される。「情報落ち」

すごく大きな数字に、すごく小さな数字をたしても、大きな数字が強すぎて小さな数字が<ruby>無視<rt>むし</rt></ruby>されてしまうことがあるのです。

たとえば、1<ruby>兆<rt>いっちょう</rt></ruby>円持っている人に1円をたしても、あまり変わらない……という感じですね。

<ViewSource path="/error/loss.ipynb" />
