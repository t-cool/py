# 忘れないようにメモをとる、かしこい作戦（動的計画法）

## 同じことを何度もやるのはもったいない！

プログラミングで、とても大切なテクニック **「<ruby>動的計画法<rt>どうてきけいかくほう</rt></ruby>」** 、かっこよく言うと **<ruby>DP<rt>ディーピー</rt></ruby>** を<ruby>紹介<rt>しょうかい</rt></ruby>しますね！

これは、一言でいうと **「計算した答えをメモしておいて、あとで使いまわす」** という作戦です。

## フィボナッチ数列でくらべてみよう

「前の２つの数字をたす」というルールのフィボナッチ数列で考えてみましょう。

ふつうのやり方（再帰）だと、こんなに大変なのです。
- `fib(5)` を知りたい！ → `fib(4)` と `fib(3)` を計算しなければいけません。
- `fib(4)` を知りたい！ → `fib(3)` と `fib(2)` を計算しなければいけません……。

あれ？ `fib(3)` を何度も計算していますよね？
これが `fib(100)` とかになると、同じ計算を<ruby>何兆回<rt>ちょうかい</rt></ruby>もすることになって、コンピュータが動かなくなってしまいます。

### かしこい DP 作戦

計算した答えをノートにメモしておきましょう。

- `fib(3)` は「2」でしたね！ → ノートに書きます。
- 次に `fib(3)` が必要になったら、計算しないでノートを見るだけです！

これだけで、100年もかかる計算が、たったの1秒で終わるようになるのです！

## お<ruby>宝<rt>おたから</rt></ruby>を詰めこもう！「ナップサック問題」

DPを使って、こんな問題も解けます。

> あなたは<ruby>冒険者<rt>ぼうけんしゃ</rt></ruby>です。重さ10kgまで入る「ナップサック」を持っています。
> 目の前には、いろんな重さと<ruby>価値<rt>かち</rt></ruby>があるお宝が並んでいます。
> どれを組み合わせたら、一番価値が高くなるでしょうか？

ひとつずつ全部の組み合わせを試すと大変ですが、**「この重さのときは、これが<ruby>最高<rt>さいこう</rt></ruby>！」** とメモしながら考えていくと、あっという間に答えが見つかります。
