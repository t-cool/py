# 計算にかかる時間はどれくらい？（計算量）

## <ruby>計算量<rt>けいさんりょう</rt></ruby>ってなに？

プログラムを作るとき、一番大切なのは「正しく動くこと」ですが、その次に大切なのが「どれくらい速く動くか」です。

データが10個くらいのときは<ruby>一瞬<rt>いっしゅん</rt></ruby>で終わっても、データが1億個になったら、100年もかかってしまうプログラムでは<ruby>困<rt>こま</rt></ruby>りますよね。

プログラムを動かすのに、どれくらい<ruby>手間<rt>てま</rt></ruby>がかかるかを **<ruby>計算量<rt>けいさんりょう</rt></ruby>** といいます。

## どれくらい<ruby>大変<rt>たいへん</rt></ruby>かを表す「<ruby>O<rt>オーダー</rt></ruby>」

データの数を $n$ 個としたとき、どれくらい大変になるかを「$O(n)$」のように表します。

- **$O(1)$**：データが何個になっても、ずっと一瞬です！
- **$O(n)$**：データの数が増えた分だけ、計算も増えます。
- **$O(n^2)$**：データが2倍になると、計算は4倍（2×2）！ 10倍になると100倍（10×10）！ どんどん大変になります。
- **$O(2^n)$**：データがちょっと増えるだけで、とんでもない時間がかかりますよ！

## 実際にくらべてみよう

Pythonでは、1秒間にだいたい $10^7$（1000万）回くらい計算ができます。

| データの数 | $O(n)$ | $O(n^2)$ | $O(2^n)$ |
| :--- | :--- | :--- | :--- |
| 10個 | 10回（一瞬） | 100回（一瞬） | 1024回（一瞬） |
| 100個 | 100回（一瞬） | 1万回（一瞬） | 地球が滅びるくらいの時間 |
| 100万個 | 100万回（一瞬） | 1兆回（3時間くらい） | むり！ |
